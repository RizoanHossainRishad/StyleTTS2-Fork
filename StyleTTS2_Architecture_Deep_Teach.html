00

MotivationBig Picture — What Problem StyleTTS2 Solves

The Core Problem: Style is Entangled With Content

Traditional TTS systems (Tacotron2, FastSpeech2) learn to map text → mel-spectrogram → waveform. They do this well for a single speaker but they conflate what is said (linguistic content) with how it is said (style: pitch contour, rhythm, energy, timbre). When I want to control style — make a sentence sound excited, slow, or like a different speaker — these models have no clean handle to pull.

StyleTTS2's central thesis: style is a continuous latent vector s that can be (a) extracted from reference audio, or (b) synthesised from text alone via a diffusion model. Once I have s, the decoder generates audio conditioned on both content and style independently.

StyleTTS2 has two separate style vectors: s_a (acoustic style — timbre, speaker identity, recording quality) and s_p (prosodic style — pitch range, speaking rate, energy pattern). They are computed by different encoders, concatenated as s_trg = [s_a ; s_p], and the diffusion model is trained to generate this concatenated vector from text context alone at inference time.

01

Style RepresentationThe Two Style Encoders

Acoustic Style Encoder → s_a model.style_encoder

Takes a mel-spectrogram segment as input (shape (1, 80, T)). Internally uses a stack of 2D convolutional layers followed by an attention pooling mechanism that collapses the time dimension into a single fixed-size vector.

What it captures: Speaker identity, vocal tract characteristics, microphone quality, room acoustics, breathiness — anything that is consistent across the whole utterance regardless of which words are spoken.

s_a = StyleEncoder(mel)   → shape: (B, d_style)
# d_style is typically 128 in the default config

In training, s_a is computed from the ground-truth mel crop of the current training sample. At inference, it comes from either a reference audio file (voice cloning mode) or from the diffusion model's output.

Prosodic Style Encoder → s_p model.predictor_encoder

Same architecture as the acoustic style encoder but trained separately to capture prosodic variation — pitch range, energy contour, speaking rate and rhythm.

Key detail: At the start of second-stage training, this module is initialised as a deep copy of the acoustic encoder's weights, then fine-tuned to specialise on prosodic features. This warm-start prevents slow convergence from random initialisation.

s_p = PredictorEncoder(mel)   → shape: (B, d_style)

s_trg = concat[s_a, s_p]  → shape: (B, 2*d_style)
# s_trg is the TARGET for the diffusion model

The concatenation s_trg = [s_a ; s_p] is the full style vector that the diffusion denoiser is trained to reproduce from text embeddings alone.

Separating acoustic and prosodic style allows independent control at inference. I can give the model a reference clip for whose voice (acoustic style) while letting the diffusion model determine how it's spoken (prosodic style) — useful for zero-shot TTS where I want a new voice but natural prosody.

02

Fundamental FrequencyPitch Extractor — JDC Net

Joint Detection and Classification Network (JDCNet) model.pitch_extractor

A pretrained neural F0 (fundamental frequency / pitch) estimator. It takes a mel-spectrogram as input and outputs three things:

  • F0_real — the ground-truth pitch contour as a continuous value per frame (in Hz or log scale)
  • uv — voiced/unvoiced flag per frame (1 = voiced, 0 = unvoiced/silence)
  • F0 — a 2D feature map that captures pitch at multiple resolutions, reshaped for downstream use
F0_real, _, F0 = pitch_extractor(gt.unsqueeze(1))

# Reshape: (B, T/2, H, 1) → (B, T, H) for decoder conditioning
F0 = F0.reshape(B, T/2 * 2, H, 1).squeeze()

Why F0 matters so much: Human perception of naturalness is disproportionately driven by pitch. A flat or incorrect pitch contour makes speech sound robotic even if the spectral quality is perfect. JDCNet was pretrained on singing voice data, giving it extremely fine-grained pitch resolution — this is why StyleTTS2 uses it rather than a simple autocorrelation-based extractor.

How it feeds the system: During training, F0_real is used both as a decoder conditioning signal (ground-truth path) and as a supervision target for the prosody predictor's F0_fake output. The L1 loss between them is loss_F0_rec.

loss_F0_rec = smooth_l1( F0_real, F0_fake ) / 10
Divided by 10 to down-weight pitch loss in the early training phase when reconstruction is still unstabilised.
03

Text ProcessingText Aligner → Acoustic Text Encoder

Text Aligner (ASR-CNN) model.text_aligner

A pretrained ASR-based convolutional model that operates on the mel-spectrogram to produce a soft alignment matrix between mel frames and phoneme tokens. It outputs three things:

  • ppgs — Phoneme Posterior Grams: probability distribution over phonemes at each frame
  • s2s_pred — seq2seq prediction logits for the text aligner's classification head
  • s2s_attn — soft attention matrix shape (B, mel_frames, n_phonemes)
ppgs, s2s_pred, s2s_attn = text_aligner(mels, mask, texts)

# Transpose & trim to get (B, T_phoneme, T_mel)
s2s_attn = s2s_attn.transpose(-1,-2)[..., 1:].transpose(-1,-2)

# Force monotonic via Viterbi (CTC-style)
s2s_attn_mono = maximum_path(s2s_attn, mask_ST)

Monotonic alignment is critical. Speech is inherently monotonic — I cannot say the third phoneme before the second. The maximum_path() function runs a Viterbi algorithm that forces the soft attention into a hard monotonic path while still preserving gradient flow for training. This is the same idea as in VITS.

Acoustic Text Encoder → h_text model.text_encoder

A Transformer-based encoder that maps phoneme tokens → contextual phoneme embeddings t_en. These embeddings capture linguistic context — the same phoneme sounds different at the start vs. end of a word, and this encoder learns that.

t_en = text_encoder(texts, input_lengths, text_mask)   → (B, d_text, T_phoneme)

# Apply alignment to get frame-level text features
asr = t_en @ s2s_attn_mono   → (B, d_text, T_mel)

The matrix multiplication t_en @ s2s_attn_mono is the alignment-weighted expansion of phoneme features into mel-frame space. Each mel frame now has a soft combination of the surrounding phoneme embeddings, weighted by how much attention the aligner placed on each phoneme at that time step. The diagram labels this as a_align × h_text.

During training, there is a 50% coin flip between using the soft attention and the monotonic attention for this multiplication — this acts as a stochastic regulariser that prevents the decoder from overfitting to a single alignment mode.

04

Linguistic ContextPL-BERT & Prosodic Text Encoder

PL-BERT (Phoneme-Level BERT) model.bert

A BERT model pretrained at the phoneme level (not word level). Standard BERT operates on WordPiece tokens; PL-BERT operates on IPA phoneme sequences, giving it deep understanding of phonetic context, stress patterns, and pronunciation ambiguities.

Why not just use regular BERT? Word-level embeddings don't resolve homographs at the phoneme level (e.g. "read" past tense vs. present tense sounds different). PL-BERT was trained with a masked phoneme prediction objective on a large corpus of G2P-converted text, so it knows the phonetic structure of language directly.

bert_dur = bert(texts, attention_mask=(~text_mask).int())   → (B, T_phoneme, d_bert)

d_en = bert_encoder(bert_dur).transpose(-1,-2)   → (B, d_enc, T_phoneme)

The bert_encoder is a lightweight projection network that maps PL-BERT's output dimension to the dimension expected by the prosody predictor. d_en is called h_bert in the diagram.

Notice there are two completely separate text encoding paths in StyleTTS2:

Path 1 (Acoustic): Phonemes → text_encoder → aligned with mel via s2s_attn → feeds the Decoder directly. This path is about what sound to produce.

Path 2 (Prosodic): Phonemes → PL-BERT → bert_encoder → feeds the Duration Predictor and Prosody Predictor. This path is about when and how to produce each sound.

The separation means the decoder never has to learn duration or pitch from scratch — it receives pre-aligned, pre-pitched content from dedicated modules.

05

Temporal ControlDuration Predictor

Duration Predictor model.predictor (duration_proj + lstm)

Predicts how many mel frames each phoneme should occupy. This determines the speaking rate — a phoneme assigned 8 frames is spoken slower than one assigned 3 frames.

Architecture: An LSTM operating on d_en (BERT features) conditioned on s_p (prosodic style). The conditioning is critical — a fast speaker style vector should produce shorter durations than a slow speaker style vector for the same text.

d, p = predictor(d_en, s_p, input_lengths, s2s_attn_mono, text_mask)

# d = duration logits (B, T_phoneme, max_dur) — decoded via sigmoid sum
# p = prosody features (B, d_prosody, T_mel) — used by F0Ntrain

Duration supervision: The ground-truth duration d_gt is derived from the monotonic alignment: d_gt = s2s_attn_mono.sum(axis=-1) — the sum of alignment weights along the mel-frame axis gives the number of frames per phoneme.

# Soft binary cross-entropy on alignment mask:
loss_ce = BCE(_s2s_pred.flatten(), _s2s_trg.flatten())

# L1 on predicted vs actual frame counts (skip start/end):
loss_dur = L1( sigmoid(_s2s_pred).sum(axis=1)[1:-1], d_gt[1:-1] )
06

Pitch & EnergyProsody Predictor — F0Ntrain

F0Ntrain — Pitch & Energy from Prosody Features model.predictor.F0Ntrain

This is a sub-module inside the predictor that takes the frame-level prosody features p_en (the cropped slice of p) and the prosodic style s_p to predict two per-frame signals:

  • F0_fake — predicted pitch contour (fundamental frequency per frame)
  • N_fake — predicted energy/normalisation per frame (log-normalised loudness)
F0_fake, N_fake = predictor.F0Ntrain(p_en, s_dur)

# Supervised by JDCNet ground truth:
loss_F0_rec = smooth_l1(F0_real, F0_fake) / 10
loss_norm_rec = smooth_l1(N_real, N_fake)

N_real is computed as log_norm(gt.unsqueeze(1)) — the log-normalised energy of the ground-truth mel. This gives the model a target for how loud each frame should be, independent of the overall amplitude.

These two signals together with the aligned text features en constitute the three conditioning inputs to the Decoder: content + pitch + energy → waveform.

y_rec = Decoder( en, F0_fake, N_fake, s_a )
The decoder's four inputs: aligned text, predicted pitch, predicted energy, and acoustic style vector.
07

Waveform GenerationThe Decoder — HiFi-GAN Based

Decoder — Conditional HiFi-GAN Generator model.decoder

The decoder is a modified HiFi-GAN generator — a stack of transposed convolutions that upsample the feature sequence from mel-frame rate (80 frames/sec) to waveform sample rate (24,000 samples/sec), a factor of 300×.

What makes it "StyleTTS2's decoder" vs. vanilla HiFi-GAN: it is conditioned on the acoustic style vector s_a via AdaIN (Adaptive Instance Normalisation) layers inserted between every upsampling block. AdaIN modulates the scale and shift of each feature map using s_a, allowing the decoder to "sound like" the target speaker at every level of the hierarchy.

# AdaIN inside each upsampling block:
h = AdaIN(h, s_a)
    = (h - mean(h)) / std(h) * γ(s_a) + β(s_a)

# γ and β are learned linear projections from s_a

F0 and N conditioning: The pitch and energy signals are injected into the lowest-level feature maps of the decoder (before the first upsample). This gives the generator precise frame-level pitch control without the global style vector needing to encode time-varying pitch information — a clean separation of concerns.

Output: A raw waveform tensor shape (B, 1, T_wave) where T_wave = T_mel × 300. At 24kHz and 80-frame mel, this is 300 samples per mel frame.

In the training code, the decoder is called in two contexts:

Ground-truth path (no grad): y_rec_gt_pred = model.decoder(en, F0_real, N_real, s) — uses real F0/N from JDCNet. This produces the "ideal" reconstruction and is used as the SLM adversarial target.

Predicted path (with grad): y_rec = model.decoder(en, F0_fake, N_fake, s) — uses predicted F0/N from the prosody predictor. This is the output that actually gets trained via all the losses.

08

Text-Driven Style GenerationStyle Diffusion Denoiser

Diffusion Denoiser — ADPM2 / Karras Schedule model.diffusion + DiffusionSampler

This is the component that makes StyleTTS2 a zero-shot TTS system. During inference, when no reference audio is available, the diffusion model generates the style vector s_trg purely from the PL-BERT text embeddings.

What kind of diffusion? StyleTTS2 uses a score-based diffusion model trained with the EDM (Elucidating Diffusion Models) framework — specifically with a Karras noise schedule and the ADPM2 sampler. The key advantage of EDM is that sigma (noise level) is treated as a continuous variable with an explicit schedule rather than a fixed discrete number of steps.

# Training: add noise to s_trg and learn to denoise
loss_diff = diffusion(s_trg.unsqueeze(1), embedding=bert_dur).mean()

# Inference: start from pure Gaussian noise, denoise conditioned on text
s_preds = sampler( noise = randn_like(s_trg).unsqueeze(1), embedding = bert_dur, embedding_scale = 1.0, embedding_mask_proba = 0.1, # CFG dropout num_steps = randint(3,5) # 3–5 ADPM2 steps ).squeeze(1)

# Supervised to match ground truth style:
loss_sty = L1(s_preds, s_trg.detach())

Classifier-Free Guidance (CFG): The embedding_mask_proba=0.1 randomly drops the text embedding during training 10% of the time, training an unconditional path alongside the conditional path. At inference, this enables guidance scale > 1 to push the style more strongly toward what the text embedding suggests.

Only 3–5 steps: Unlike image diffusion models that need 20–1000 steps, the style vector is low-dimensional (~256 floats) so the ADPM2 sampler converges in 3–5 steps. This makes inference fast.

Sigma estimation: If estimate_sigma_data=True, the model tracks the running standard deviation of style vectors and updates sigma_data accordingly. This is the EDM normalisation constant — getting it right dramatically improves diffusion training stability.

At Training Time

Style vector s_trg is extracted from ground-truth mel. Diffusion trains to reconstruct it from noise + text. Loss: loss_diff (score matching) + loss_sty (L1 on sampled output).

At Inference Time

No reference audio needed. Sample s_preds from Gaussian noise conditioned on PL-BERT embeddings of the input text in 3–5 ADPM2 denoising steps.

09

Perceptual QualityMPD & MRD Discriminators

Multi-Period Discriminator (MPD) + Multi-Resolution Discriminator (MRD) model.mpd + model.msd

Two families of discriminators working in parallel, operating in the waveform domain (not mel domain). Their job is to force the decoder to produce waveforms that are perceptually indistinguishable from real recordings.

DiscriminatorWhat It SeesWhat It Catches
MPD (Multi-Period) Waveform reshaped into 2D arrays at periods p ∈ {2,3,5,7,11} — prime numbers to avoid aliasing Periodic artefacts — buzzing, roughness, unnaturally repeating micro-patterns in voiced sounds
MRD (Multi-Resolution) STFT spectrograms at multiple (n_fft, hop, win) combinations Spectral artefacts — blurriness, missing overtones, incorrect formant structure

GAN update order in code:

# Step 1: Train discriminator on real vs. detached fake
d_loss = dl(wav.detach(), y_rec.detach()).mean()
d_loss.backward()
optimizer.step('msd'); optimizer.step('mpd')

# Step 2: Train generator to fool discriminator
loss_gen_all = gl(wav, y_rec).mean()
# loss_gen_all is part of g_loss, backpropagated through decoder

The .detach() in the discriminator update is the key GAN training invariant — it ensures the discriminator gradient does not flow back into the generator. Without it, the generator and discriminator would interfere with each other's updates.

WavLM Discriminator (SLM Adversarial) — Phase 3 Only model.wd + slmadv

A third discriminator activated only after joint_epoch. Instead of raw waveforms, it operates in the WavLM feature space — a self-supervised speech representation model pretrained on 94,000 hours of speech.

Why WavLM features? WavLM captures linguistic and prosodic naturalness that raw waveform discriminators cannot. If the generated speech has correct phonemes and pitch but unnatural rhythm or coarticulation, WavLM will detect it. This is the loss that most directly improves perceived naturalness in listening tests.

Gradient safety: The SLM gradients can be very large. The code normalises them by the predictor's L2 norm and applies an additional scale factor to the duration and LSTM modules to protect them from the SLM's aggressive updates.

10

Curriculum LearningThe Three-Phase Training Schedule

Phase 1 — epoch 0 → diff_epoch Reconstruction + Alignment + GAN

Train the acoustic pipeline end-to-end: text aligner, text encoder, predictor (F0 + duration), decoder, and MPD/MRD discriminators. The diffusion module is frozen / inactive. Style vectors come directly from the style encoders applied to ground-truth mels.

Active losses: loss_mel, loss_F0_rec, loss_norm_rec, loss_dur, loss_ce, loss_gen_all, loss_lm, loss_mono, loss_s2s

Phase 2 — diff_epoch → joint_epoch Diffusion Head Activates

The diffusion denoiser starts training. The sampler is used to generate style predictions during the forward pass, and both loss_diff (score matching) and loss_sty (L1 style matching) become active. The sigma_data estimate is updated each step.

New losses: + loss_diff, + loss_sty

Phase 3 — joint_epoch → end SLM Adversarial Loss Activates

The WavLM-based adversarial training loop activates. The slmadv module generates new samples, computes WavLM feature distances, and trains both the WavLM discriminator and the generator adversarially. Gradient normalisation is applied to protect sensitive modules.

New losses: + d_loss_slm, + loss_gen_lm

Training all losses simultaneously from epoch 0 would fail. The GAN needs the generator to first produce reasonable outputs before the discriminator can give useful gradient signal. The diffusion model needs the style encoders to produce consistent vectors before it can learn to match them. The SLM adversarial loop needs the decoder to produce intelligible speech before WavLM features are meaningful. The curriculum is not optional — it's the reason StyleTTS2 trains stably at all.

End-to-EndInference Pipeline

1. Text → Phonemes → PL-BERT

Input text is converted to IPA phoneme sequence via G2P. PL-BERT produces contextual phoneme embeddings bert_dur. The text encoder produces t_en.

2. Style Generation — Diffusion (if no reference) OR Style Encoder (if reference given)

Zero-shot: Sample s_trg from diffusion model conditioned on bert_dur in 3–5 ADPM2 steps. Split into s_a, s_p.
Voice cloning: Extract s_a = style_encoder(ref_mel), s_p = predictor_encoder(ref_mel) directly.

3. Duration Prediction → Alignment Expansion

Duration predictor uses d_en and s_p to predict frame counts per phoneme. The text features t_en are expanded to mel-frame rate via the predicted alignment.

4. F0 & Energy Prediction

F0Ntrain(p_en, s_p) produces per-frame pitch and energy contours conditioned on prosodic style. These are deterministic given the style and content — no sampling.

5. Decoder → Waveform

decoder(en, F0_fake, N_fake, s_a) upsamples 300× from mel-frame rate to 24kHz waveform. AdaIN layers throughout shape the timbre to match s_a. Output is a raw PCM waveform.

StyleTTS2's biggest architectural advance over both VITS and StyleTTS1 is the disentangled dual style space + text-conditioned diffusion. VITS cannot generate a new speaker style without a reference. StyleTTS1 has a style encoder but no way to synthesise style from text alone. StyleTTS2 is the first system where I can give it only text and get back speech that sounds like a coherent, natural voice — because the diffusion model has learned the distribution of natural speaking styles conditioned on what is being said.
